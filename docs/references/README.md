# Reference Registry

`references.yaml` is the source of truth. The registry records stable `REF-NNN`
source IDs, DOI/authors, roles, topics, and verified pull-quotes where available;
the generated HTML and `cited_by` index make model-bearing numbers auditable
against the literature. The HTML files are generated by:

```shell
.venv/bin/python docs/references/build_references.py
```

Use `--check` for validation only:

```shell
.venv/bin/python docs/references/build_references.py --check
```

## REF IDs

Each source gets one global, stable ID: `REF-001`, `REF-002`, and so on. IDs
must be zero-padded and must not be reused for a different source.

## Citing Model Inputs

Put the reference token in existing source strings or code comments.

YAML example:

```yaml
source: "REF-001 Kress & Carmichael 1991; ferric/ferrous redox model"
```

Python comment example:

```python
# REF-001 Kress & Carmichael 1991 redox basis.
```

The generator scans model-bearing source files and computes `cited_by`
automatically. Do not hand-edit `cited_by` into `references.yaml`.

## Adding A Reference

Add a new entry under `references:` in `references.yaml`.

Required fields: `authors`, `title`, `year`, `role`, `topic`, `status`, `doi`,
`url`, `found`, and `pull_quotes`.

`role` must be one of: `DATA`, `MODEL`, `MEASUREMENT`, `THEORY`, `REVIEW`,
`OBSERVATION`, `TEXTBOOK`.

`topic` is a kebab-case page group, used for `topics/<topic>.html`.

`pull_quotes` must contain only verified verbatim source passages. If no quote
has been verified, leave `pull_quotes: []` and set `needs_quote: true`. Never
invent a quote or DOI.

## Supersession

To mark a source as superseded, set:

```yaml
status: superseded_by:REF-123
```

The generator surfaces the `cited_by` list for superseded references so model
numbers depending on outdated sources are visible.

## Generated Files

Generated HTML:

- `index.html`
- `topics/<topic>.html`

These files are static and browser-openable. Re-run the generator after every
registry or source-token change.
